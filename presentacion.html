1. Diseño del Prototipo

1.1. Boceto inicial o prototipado

¿Qué queríamos validar?

El boceto inicial buscaba validar una interfaz de POS que unificara el cobro, el despacho y la reposición en una sola vista.

  • Reducción de clics para aplicar promociones.
  • Visibilidad inmediata del stock en almacén.
  • Botones táctiles grandes para evitar errores por fatiga.
1.2 Características Clave

TamboSync Features

Funcionalidades diseñadas para resolver los dolores de "Carlos El Multifunción".

1. Smart Combo

Autodetección de promociones al pasar el escáner. Validación automática de descuentos sin cálculos mentales.

2. Stock-Sync

Indicador visual en pantalla del stock real en almacén. Evita viajes innecesarios y reduce tiempos de espera.

3. Pago OneTouch

Integración nativa de Yape y Plin en la terminal de cobro. Aprobación en 1 clic sin usar un POS físico extra.

4. Night Mode

Interfaz de alto contraste optimizada para proteger la vista del colaborador durante los turnos de madrugada.

4.1 Análisis del Feedback

Conclusiones de la Prueba

¿Qué funcionó bien?

  • El login con PIN fue validado como ágil para la rápida rotación en tienda.
  • La UI con botones grandes eliminó los errores táctiles bajo estrés.
  • La sugerencia de combos básica redujo notablemente la carga mental del cajero.

Problemas detectados

  • Imposibilidad de anular un producto individual (solo se podía limpiar todo el carrito).
  • El sistema ignoraba combos complejos (de más de 2 productos).
  • Cero opciones para cobrar extras rápidos como las bolsas ecológicas.

Evidencias de mejora

Notamos una gran frustración en los colaboradores cuando cometían un error mínimo y el sistema les obligaba a borrar toda la venta para empezar de cero.

Además, el retraso al intentar aplicar promociones de varios productos a mano causaba fricción en la cola de espera.

4.2 Ajustes Realizados

Mejoras al Prototipo

Prototipo UI Mejorado

1. Borrado Individual

Se integró un icono de eliminación [X] por cada línea en el ticket de compra.

✅ Resuelve: La obligación de reiniciar todo el escaneo tras un error manual.

2. Lógica Smart Combo V2

El algoritmo se reescribió para detectar combinaciones de 3 o más productos simultáneamente.

✅ Resuelve: Pérdida de tiempo y fricción al cobrar promociones complejas.

3. Botones Rápidos

Panel directo en pantalla para agregar "Bolsa Ecológica" o "Cubiertos" con un toque.

✅ Resuelve: El bloqueo al intentar cobrar artículos no escaneables.

5.1 Segunda Ronda de Pruebas

Validación de Mejoras

Sometimos el prototipo actualizado a una simulación de hora punta.

¿Qué se probó?

Los usuarios recrearon el flujo crítico de cobro forzando errores intencionales:

  • Equivocarse al escanear y usar el nuevo botón [X].
  • Cobrar un mix de cervezas, snacks y hielos.
  • Añadir bolsas de plástico a último segundo.

Mejoras Confirmadas

  • Cero reinicios: El borrado individual salvó la venta sin causar frustración.
  • Agilidad: El botón de "Bolsa" redujo un paso molesto a solo 1 clic.
  • Confianza: Ver el descuento aplicado en combos complejos dio seguridad plena al cajero.

Aún Persisten

  • Integración de Pasarelas: La validación de tarjetas requiere confirmación visual externa; falta enlace directo con el terminal Izipay.
  • Sincronización ERP: El stock se descuenta bien a nivel local, pero falta automatizar la conexión con el inventario maestro central.
5.2 Evaluación de Proyecto

Viabilidad y Factibilidad

a. Viabilidad de Negocio

¿Es posible llevarlo a la realidad? Sí, es altamente viable.

  • Recursos: Requiere pantallas táctiles estándar en cajas, hardware que las tiendas Tambo actuales pueden soportar.
  • Tiempo: Un MVP funcional para tiendas piloto tomaría entre 3 a 4 meses de desarrollo ágil.
  • Alcance: Se limitará inicialmente solo al cobro rápido y control de stock local en tienda física.
  • Complejidad: El mayor reto será la integración con las bases de datos Legacy (inventario maestro).

b. Factibilidad Técnica

¿Contamos con la capacidad para implementarlo? Sí.

  • Conocimientos: El equipo domina el stack propuesto: React para la vista del POS y Spring Boot para la API.
  • Herramientas: Contamos con entornos de nube (AWS/GCP) y WebSockets para la sincronización de stock al instante.
  • Capacidades Externas: Será imperativo requerir licencias de SUNAT (Facturación) y las APIs privadas de Niubiz/Izipay.
